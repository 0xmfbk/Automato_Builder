Features & Workflow

  • Add State: Click on empty canvas space (not on a transition) to add a new state (auto-named q0, q1, ...).
  • Move State: Drag a state to reposition it. States snap to a grid and cannot overlap.
  • Rename State: Double-click a state (on canvas or in the sidebar) to rename. Names must be unique.
  • Set Start State: Select a state, then click Set Start in the sidebar, or activate the button and click a state.
  • Toggle Accept State: Select a state, then click Toggle Accept in the sidebar, or activate the button and click a state.
  • Delete State: Select a state, then click Delete State in the sidebar, or activate the button and click a state.
  • Add Transition: Select a source state, click Add Transition, then click the target state. Enter the transition symbol (comma-separated for multiple, use ε for epsilon transitions).
  • Delete Transition: Click Delete Transition, then click the transition label you want to remove.
  • Simulate Input: Enter an input string in the sidebar. Click Simulate to run, or Step to step through one symbol at a time. Current states and transitions are highlighted. The full path is shown after simulation. Click Reset to clear.

Automata Types

  • DFA: Deterministic Finite Automaton (one transition per symbol per state).
  • NFA: Nondeterministic Finite Automaton (multiple transitions per symbol allowed).
  • ε-NFA: NFA with epsilon (empty string) transitions. Use ε as a symbol.

Canvas & Sidebar Controls

  • Pan Canvas: Right-click and drag on empty canvas space to move the entire automaton.
  • Responsive Canvas: The canvas automatically resizes to fill available space as you resize the window or device.
  • Sidebar: All controls are in the right sidebar (always visible on desktop, togglable on mobile). Use the sidebar for state/transition management, simulation, and import/export.
  • Tooltip: Hover over states or transitions to see their names/symbols.

States

  • Start State: Marked with a blue arrow from the right.
  • Accept State: Marked with a double circle and green border.
  • Selected State: Highlighted in orange.
  • Sidebar List: All states are listed in the sidebar for quick selection and renaming.

Transitions

  • Add: Select a source state, click Add Transition, then click the target state. Enter the symbol(s).
  • Self-loops: Supported and drawn as circles above the state.
  • Multiple Transitions: Multiple transitions between the same two states are shown as separate curves.
  • Flex/Curve: Hover over a transition to show a blue handle. Drag the handle to flex the curve at any time.
  • No Duplicates: You cannot add two transitions with the same symbol from the same state.
  • Delete: Click Delete Transition, then click the transition label to remove it.

Simulation

  • Input String: Enter a string in the sidebar input box.
  • Simulate: Click Simulate to run the automaton on the input string.
  • Step: Click Step to step through the input one symbol at a time.
  • Reset: Click Reset to clear the simulation and start over.
  • Highlighting: The current state(s) and transition are highlighted during simulation.
  • Path: The full simulation path is shown after running.

Import, Export & Save

  • Export JSON: Save your automaton as a JSON file for backup or sharing.
  • Import JSON: Load a previously saved automaton from a JSON file (must be from this tool).
  • Export Image: Save the automaton diagram as a PNG image.
  • Clear: Remove all states and transitions to start fresh.

Shortcuts & Pro Tips

  • Esc or Right-click to cancel adding a transition.
  • Double-click a state to rename it.
  • Right-click and drag on empty canvas to pan the automaton.
  • Hover over a transition to flex its curve.
  • Click a state to select and edit.
  • Automaton Type: Changing the automaton type only affects simulation rules; it does not convert your automaton.
  • Best Practice: Use unique, short state names and keep your automaton organized for easier debugging and sharing.
  • Tip: Export your automaton regularly as JSON or PNG to avoid data loss.
Advanced: You can always flex transition curves, pan the canvas, and edit states/transitions at any time. The UI is fully responsive and touch-friendly on mobile devices.

FAQ

Can I create self-loops and multiple transitions?

Yes! Self-loops are supported and drawn as circles above the state. Multiple transitions between the same two states are shown as separate curves, as long as their symbols are unique from the same source.

How do I pan or move the whole automaton?

Right-click and drag on empty canvas space to move the entire automaton diagram.

How do I cancel adding a transition?

Press Esc or right-click while the orange line is active to cancel transition-adding mode.

Can I import automata from other tools?

You can import automata saved as JSON from this tool. Support for other formats may be added in the future.

What does changing the automaton type do?

It only changes the simulation rules (DFA/NFA/ε-NFA). It does not convert your automaton or its transitions.
